# Java StringUtils Example

This is an example Java Maven project demonstrating the Claude Code Automated Testing Plugin's capabilities with JUnit 5.

## Project Overview

A simple utility class (`StringUtils`) with common string manipulation methods. This project has NO tests initially - they will be generated by the Claude Code testing plugin.

## StringUtils Methods

The `StringUtils` class provides the following utility methods:

- `capitalize(String)` - Capitalizes the first letter of a string
- `reverse(String)` - Reverses a string
- `isPalindrome(String)` - Checks if a string is a palindrome
- `countVowels(String)` - Counts vowels in a string
- `repeat(String, int)` - Repeats a string N times
- `truncate(String, int)` - Truncates a string with ellipsis
- `isAlphabetic(String)` - Checks if string contains only letters
- `removeWhitespace(String)` - Removes all whitespace
- `countWords(String)` - Counts words in a string
- `toTitleCase(String)` - Converts to title case

## Maven Configuration

### Dependencies

- **JUnit 5 (Jupiter)**: 5.10.1
  - `junit-jupiter-api` - API for writing tests
  - `junit-jupiter-engine` - Test engine for running tests
  - `junit-jupiter-params` - Parameterized test support

### Plugins

- **Maven Compiler Plugin** (3.11.0): Compiles Java 11 source
- **Maven Surefire Plugin** (3.2.2): Runs unit tests
  - Naming patterns: `*Test.java`, `Test*.java`, `*Tests.java`
- **JaCoCo Plugin** (0.8.11): Code coverage reporting (optional)

## Maven Commands

### Compile the Project

```bash
mvn compile
```

This compiles the source code in `src/main/java`.

### Run Tests

```bash
mvn test
```

This compiles and runs all tests in `src/test/java` matching the Surefire naming patterns.

### Clean and Build

```bash
mvn clean install
```

This cleans previous builds, compiles, runs tests, and packages the JAR.

### Run Specific Test Class

```bash
mvn test -Dtest=StringUtilsTest
```

Replace `StringUtilsTest` with your test class name.

### Generate Coverage Report

```bash
mvn test
```

JaCoCo generates coverage report at: `target/site/jacoco/index.html`

## Using Claude Code Testing Plugin

### Step 1: Analyze the Code

```
/test-analyze examples/java-string-utils
```

This will:
- Detect Java language and Maven build system
- Identify JUnit 5 as the testing framework
- Analyze `StringUtils.java` methods
- Generate a test plan with priorities

### Step 2: Generate Tests

```
/test-generate
```

This will:
- Create a comprehensive test plan (10 methods to test)
- Generate JUnit 5 test classes
- Include parameterized tests, edge cases, and error conditions
- Place tests in `src/test/java/com/example/`

### Step 3: Execute Tests

After approving the generated tests, the plugin will automatically:
- Run `mvn test`
- Parse Maven Surefire output
- Display pass/fail results
- Provide coverage metrics

### Step 4: Human-in-the-Loop Workflow

```
/test-loop examples/java-string-utils
```

This interactive workflow:
- Analyzes code
- Generates test plan (approval gate #1)
- Generates test code (approval gate #2)
- Executes tests
- Validates results (approval gate #3)
- Iterates on failures if needed

## Expected Test Generation

The plugin should generate tests covering:

1. **Happy Path Tests**
   - `testCapitalizeNormalString()`
   - `testReverseNormalString()`
   - `testIsPalindromeTrue()`

2. **Edge Cases**
   - `testCapitalizeEmptyString()` - Expects IllegalArgumentException
   - `testReverseNull()` - Returns null
   - `testCountVowelsEmptyString()` - Returns 0

3. **Error Conditions**
   - `testIsPalindromeNull()` - Expects IllegalArgumentException
   - `testRepeatNegativeCount()` - Expects IllegalArgumentException
   - `testTruncateInvalidMaxLength()` - Expects IllegalArgumentException

4. **Parameterized Tests**
   - Multiple palindrome test cases (radar, level, A man a plan...)
   - Multiple vowel counting test cases
   - Title case with various inputs

## Troubleshooting

### Maven Not Found

```bash
# Check if Maven is installed
mvn --version

# If not installed, download from:
# https://maven.apache.org/download.cgi
```

### Tests Not Found

Ensure test files match Surefire naming patterns:
- `*Test.java` (e.g., `StringUtilsTest.java`)
- `Test*.java` (e.g., `TestStringUtils.java`)
- `*Tests.java` (e.g., `StringUtilsTests.java`)

### Compilation Errors

```bash
# Clean and recompile
mvn clean compile test-compile
```
